# nishikama-transit-map

鎌倉市西部（腰越・深沢）と藤沢市東部（片瀬）を対象に、公共交通の利便性と地形を重ねて確認できるウェブ地図です。

大船駅・鎌倉駅・藤沢駅への直通本数、バス停・駅の時刻表、標高、徒歩・車のルートに沿った高低差を、ひとつの画面で確認できます。

**公開ページ:** [鎌倉西部・藤沢東部 地形 × 3拠点アクセスマップ](https://nishikama-ra.github.io/nishikama-transit-map/)

## 利用者向け操作案内

### 1. 3拠点への行きやすさを見る

画面右側（スマートフォンでは地図の下）の「3拠点への直通アクセス表示」で、表示対象を選びます。

- **合計**: 大船・鎌倉・藤沢の3駅へ乗り換えなしで行ける本数の合計
- **大船へ / 鎌倉へ / 藤沢へ**: 選んだ駅へ直通する本数
- **平日日中 / 平日朝**: 時間帯の切り替え

地図上の丸は、色が濃く大きいほど直通本数が多いことを示します。丸をクリックすると、停留所・駅ごとの直通本数、系統、標高などを確認できます。

### 2. 時刻表を開く

地図上のバス停または駅をクリックし、ポップアップ内の時刻表リンクを選ぶと、交通事業者の公式時刻表を別画面で開けます。

一部の停留所は、公式サイト側の仕様や運行状況によりリンクがない場合があります。最新の運行時刻は、必ず各交通事業者の公式情報で確認してください。

### 3. 系統別の本数を見る

「系統別の本数」には、平日の日中・朝の毎時本数、1日の片方向の本数、始発・終発の目安、直通する拠点を表示します。

本数や時刻はダイヤ改正で変わります。地図は比較・概観のために使い、実際の乗車前には公式時刻表を確認してください。

### 4. 地図の表示を切り替える

「レイヤ」で次の表示を個別に切り替えられます。

- 停留所・駅のアクセス表示
- 鉄道・モノレール路線
- 標高の色分け（不透明度を調整可能）
- 陰影起伏
- 地区の範囲

地図左上のレイヤ選択では、淡色地図と航空写真を切り替えられます。

### 5. 標高を調べる

「標高ツール」の **クリックで標高** を選び、地図上の地点をクリックすると、その地点のおおよその標高を表示します。

### 6. 高低差の断面図を見る

1. **断面図モード**を選びます。
2. **直線 / 徒歩 / 車**から経路の種類を選びます。
3. 地図上で始点、終点の順にクリックします。さらに地点を追加して経路を延長することもできます。
4. 距離、最低・最高標高、累積上り・下り、50m平均最大勾配を確認します。
5. 断面図上をクリックすると、対応する位置を地図上に表示します。

**徒歩**と**車**は Mapbox Directions API を使用します。距離や地点数には制限があり、通信状況や外部サービスの状態によって取得できない場合があります。

## この地図について

この地図は、地域の公共交通と地形を比較・検討するための参考資料です。交通事業者や自治体が提供する公式の経路検索・運行案内ではありません。

主な対象は、腰越・深沢・片瀬地区のバス停と鉄道駅です。大船・鎌倉・藤沢への直通アクセスを、平日の本数を基に可視化しています。

## 開発者向け情報

### 構成

ビルド工程を持たない、単一ページの静的ウェブアプリです。`main` ブランチの内容を GitHub Pages で公開します。

- 地図表示: [Leaflet 1.9.4](https://leafletjs.com/)
- 背景地図・航空写真・陰影起伏・標高: [国土地理院タイル](https://maps.gsi.go.jp/development/ichiran.html)
- 徒歩・車の経路: [Mapbox Directions API](https://docs.mapbox.com/api/navigation/directions/)
- ホスティング: GitHub Pages
- 実装: HTML / CSS / JavaScript（`index.html` に集約）

外部ライブラリと地図・標高・経路データを実行時に取得するため、利用時にはインターネット接続が必要です。

### ファイル構成

| ファイル | 用途 |
| --- | --- |
| `index.html` | 画面、地図データ、表示ロジック、標高・経路機能を含むアプリ本体 |
| `routes_data.csv` | 系統別本数データの編集・外部管理用CSV |
| `tools/export_routes_csv.mjs` | `index.html` 内の系統データをCSVへ書き出す補助ツール |
| `.nojekyll` | GitHub Pages で Jekyll 処理を無効化 |
| `AGENTS.md` | ブランチ運用、Mapbox token、実装上の方針 |

### ローカルでの確認

静的ファイルのため、簡易HTTPサーバーでリポジトリのルートを配信して確認できます。例えば Node.js が利用できる環境では次のように起動します。

```powershell
npx serve .
```

表示されたローカルURLをブラウザで開いてください。ファイルを直接開くと、ブラウザのセキュリティ制限により外部データの取得に失敗することがあります。

### 本数データの更新

表示の基礎データは `index.html` 内の系統マスタにあります。`routes_data.csv` は、系統ID、表示名、日中・朝の毎時本数、1日本数、始発・終発、直通拠点を管理するための表形式データです。

`SHEET_CSV_URL` に公開CSVのURLを設定すると、起動時に外部データを読み込みます。未設定または読み込み失敗時は、`index.html` 内の値を使用します。公開CSVはURLを知る人が閲覧できるため、公開情報だけを置いてください。

時刻表や便数を更新するときは、次を確認します。

1. 交通事業者の公式時刻表を確認する。
2. 系統別本数と停留所・駅の関連付けを更新する。
3. 時刻表リンクが正しい公式ページを開くことを確認する。
4. 平日日中・平日朝・1日計と始発終発の表示を確認する。
5. 変更後の画面をPC幅とスマートフォン幅で確認する。

### Mapbox token

徒歩・車ルート用の token は `index.html` の `MAPBOX_TOKEN` 1か所で管理します。

- このアプリ専用の public token を使用する。
- secret token は使用しない。
- public token はブラウザから見える前提で扱う。
- 公開ページ `https://nishikama-ra.github.io/nishikama-transit-map/` へのURL制限を設定する。

### ブランチと公開

`main` は GitHub Pages の公開用安定版です。リポジトリの `AGENTS.md` に従い、機能追加は `feature/...`、軽微な修正は `fix/...` ブランチで行い、確認後に `main` へ反映します。

変更時は、公開前に少なくとも次を確認します。

- 3拠点と時間帯の切り替え
- 停留所・駅のポップアップと公式時刻表リンク
- 各レイヤの表示切り替え
- 標高クリック
- 直線・徒歩・車の断面図
- 断面図と地図上の位置の連動
- 開発者ツールに重大なエラーが出ていないこと

## データと免責

停留所・鉄道・地形・本数データの出典と基準時点は、公開画面下部の「データについて」を参照してください。データは加工・集計を含み、実際の運行や地形を完全に保証するものではありません。

正確な時刻、運休、乗り場、運賃、経路は、各交通事業者の公式案内を確認してください。
